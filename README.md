# 班級出缺席管理系統

一套給導師 / 任課老師使用的班級出缺席登錄工具。前端為單一 HTML 檔（可放在 GitHub Pages），後端使用 Google Apps Script 串接 Google Sheets 作為資料庫，免自架伺服器、資料即時同步。

## 功能特色

- **快速登錄**：左側點選學生、右側點假別，一鍵完成並自動跳到下一位。
- **複數天數登錄**：選一名學生與假別後，在日曆上一次勾選多天（可跨月），一次送出。
- **批次編輯**：歷史紀錄可多選，一次修改假別或一次刪除。
- **雲端名單同步**：從「學生名單」工作表下載班級名單，並快取在瀏覽器 `localStorage`。
- **當日狀態顯示**：切換日期後自動同步該日已登錄的假別，避免重複登錄。
- **歷史紀錄檢視**：可依日期 / 座號 / 姓名 / 假別排序，並依學生篩選，支援單筆編輯與刪除。
- **一人一日一筆**：後端以 upsert 處理，同一天同座號重複登錄只會更新該列、不產生重複資料。
- **開啟即顯示**：啟動時先用上次的資料立即渲染，再於背景更新，不必空等。
- **手機友善**：響應式版面，手機上維持左右兩欄操作。

支援的假別：病假、事假、公假、喪假、生理假、曠課，以及「銷假 / 清除」。

## 系統架構

```
使用者瀏覽器 (index.html)
        │  fetch (GET / POST, JSON)
        ▼
Google Apps Script Web App (Code.gs)
        │  SpreadsheetApp
        ▼
Google Sheets（出缺席紀錄 + 學生名單）
```

- **前端**：`index.html`，原生 HTML / CSS / JavaScript，無框架、無外部相依套件。
- **後端**：`Code.gs`，部署為 Apps Script 網頁應用程式（Web App），提供 `doGet` / `doPost`。
- **資料庫**：同一份 Google 試算表中的兩個工作表。

## Google Sheets 設定

同一份試算表需包含以下兩個工作表：

### 1. 出缺席紀錄（第一個工作表）

後端固定寫入「第一個」工作表（`getSheets()[0]`），欄位順序如下：

| 欄 | 內容 | 範例 |
| --- | --- | --- |
| A | 時間戳記（系統自動填入） | 2025-09-01 08:12:33 |
| B | 日期 | 2025-09-01 |
| C | 座號 | 15 |
| D | 姓名 | 王小明 |
| E | 假別 | 病假 |
| F | 備註 | （可空白） |

> 建議把 B 欄整欄格式設為「純文字」，讓日期固定以 `yyyy-mm-dd` 儲存，資料更一致。（後端已容錯不同格式，不設定也可正常運作。）

### 2. 學生名單

工作表名稱必須是 **`學生名單`**，欄位：

| 欄 | 內容 |
| --- | --- |
| A | 座號 |
| B | 姓名 |

## 部署步驟

### 後端（Google Apps Script）

1. 開啟資料所在的 Google 試算表，點「擴充功能 → Apps Script」。
2. 將 `Code.gs` 內容整段貼上並儲存。
3. 點「部署 → 新增部署作業 → 網頁應用程式」。
4. 設定：
   - **執行身分**：我（你的帳號）
   - **誰可以存取**：所有人
5. 部署後複製「網頁應用程式 URL」。

### 前端（index.html）

1. 打開 `index.html`，把最上方的 `scriptURL` 換成上一步取得的網址：

   ```js
   const scriptURL = 'https://script.google.com/macros/s/XXXXXX/exec';
   ```

2. 將 `index.html` 放到任意靜態空間即可使用（例如 GitHub Pages）。

> **更新後端後務必重新部署**：修改 `Code.gs` 後要到「部署 → 管理部署作業 → 編輯 → 版本選『新版本』→ 部署」，否則新功能不會生效。網址不會改變，前端無需調整。

## 使用說明

介面分為三個分頁。

### 📝 紀錄出缺席

1. 選擇日期（預設為今天）。
2. 首次使用點「☁️ 從雲端同步名單」下載班級名單。
3. 從左側點選學生，再點右側假別按鈕即可登錄，畫面會自動跳到下一位。
4. 要取消某位學生當日紀錄，點該生後按「❌ 銷假 / 清除」。

### 🗓️ 複數天數紀錄

適合一次登錄長假（例如請一週病假）。

1. 從下拉選單選擇學生，並選擇假別。
2. 在日曆上點選要登錄的日期，可複選；再點一次可取消。用 `‹` `›` 切換月份，跨月的選取會保留。
3. 下方顯示「已選 N 天」，確認後按「✅ 送出登錄」，一次請求完成全部天數。

### 📊 檢視歷史紀錄

1. 點欄位標題可排序，右上可依學生篩選。
2. 單筆操作：每列可「編輯」（修改日期 / 假別）或「刪除」。
3. **批次操作**：勾選左側核取方塊（表頭可全選），上方會出現工具列，可一次「批次改假別」或「批次刪除」，不論幾筆都只送出一次請求。

## 後端 API 說明

所有回應皆為 JSON，格式為 `{ "status": "success" | "error", ... }`。

### `GET`

| 參數 | 說明 |
| --- | --- |
| `?action=getAll` | 一次回傳學生名單與出缺席紀錄（前端啟動時使用） |
| `?action=getStudents` | 只讀取「學生名單」工作表 |
| （無參數） | 只讀取第一個工作表（出缺席紀錄） |

### `POST`（body 為 JSON）

| action | 用途 | 主要參數 |
| --- | --- | --- |
| `add`（預設） | 新增或更新（upsert）某生某日假別 | `date`, `studentId`, `studentName`, `leaveType`, `note` |
| `addBatch` | 一次登錄同一位學生的多個日期 | `studentId`, `studentName`, `leaveType`, `note`, `dates`（陣列） |
| `deleteByStudentDate` | 一次清除某座號某日所有紀錄 | `date`, `studentId` |
| `edit` | 依列號修改日期與假別 | `row`, `date`, `leaveType` |
| `delete` | 依列號刪除單列 | `row` |
| `editBatch` | 一次修改多列的假別 | `rows`（陣列）, `leaveType` |
| `deleteBatch` | 一次刪除多列 | `rows`（陣列） |

`editBatch` / `deleteBatch` 的 `rows` 需帶入驗證資訊：

```json
"rows": [
  { "row": 5, "date": "2025-09-01", "seat": "15" },
  { "row": 8, "date": "2025-09-03", "seat": "15" }
]
```

回應會包含 `applied`（實際處理筆數）與 `skipped`（驗證不符而略過的筆數）。

## 技術重點

### 資料正確性

- **Upsert 去重**：後端以「日期值」比對（容忍 `2025-01-15` 與 `2025/1/15`），同日同座號存在即整列更新、其餘重複列一併刪除，可自動修復舊資料殘留的重複。
- **批次操作驗證**：`editBatch` / `deleteBatch` 以列號定位，但列號可能因資料變動而失效。送出時一併帶入該列的日期與座號，後端逐列比對相符才執行，不符者略過並回報，避免誤改他人紀錄。
- **並發保護**：`doPost` 以 `LockService` 加鎖，避免同時寫入造成競爭與重複。
- **座號容錯**：比對時容忍 `5` 與 `05` 等差異。

### 效能設計

- **批次只讀一次表**：`addBatch` 讀取一次試算表後於記憶體算出所有變更，再一次寫回。30 天的登錄由「30 次讀取 + 30 次寫入」降為「1 次讀取 + 1 次寫入」。
- **批次編輯 1 讀 1 寫**：`editBatch` 整欄讀取、改完一次寫回，與筆數無關。
- **連續列合併刪除**：`deleteBatch` 將連續的列合併為單次 `deleteRows` 呼叫。
- **伺服器端快取**：以 `CacheService` 快取讀取結果（預設 300 秒，可調整 `CACHE_TTL_SEC`），任何寫入都會清除快取。因單一值有大小上限，實作為分段存放。
- **精確範圍讀取**：以 `getRange(1, 1, getLastRow(), n)` 取代 `getDataRange()`，避免讀到殘留格式造成的多餘範圍。
- **合併啟動請求**：`getAll` 一次取回名單與紀錄，省下前端啟動時的一次往返。
- **先顯示再更新**：前端將資料存於 `localStorage`，開啟時立即渲染舊資料，再於背景更新。
- **前端渲染**：歷史表格一次組字串渲染，避免迴圈中反覆 `innerHTML +=`。

### 安全性

- 以事件委派與 `data-*` 屬性取代內嵌 `onclick` 字串，並對輸出做 HTML 跳脫，避免姓名含特殊字元造成問題。
- 操作進行中鎖定按鈕，防止連點造成重複送出。

## 注意事項

- 前端 `scriptURL` 會公開於原始碼中，屬此架構的固有特性；請視資料敏感度自行評估存取權限。
- 後端快取預設 300 秒，若在**另一台裝置**變更資料，本機最多可能延遲該時間才看到。按「🔄 載入歷史資料」可強制取得最新；亦可調小 `Code.gs` 中的 `CACHE_TTL_SEC`。
- 啟動時會自動更新學生名單。若雲端名單讀取為空則不覆蓋本機資料，以免誤刪。
- 名單快取存於瀏覽器 `localStorage`（鍵值 `classStudentList`），紀錄快取鍵值為 `cachedRecords`；更換裝置或清除瀏覽器資料後需重新同步。
- 批次操作若出現「部分略過」提示，代表資料已被變更，畫面會自動重新整理，請確認後再操作一次。
